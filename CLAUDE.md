# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

This is a shared utilities repository for MkDocs-based documentation sites. The files here are meant to be copied or symlinked into consumer MkDocs projects, not run directly from this repo.

## Structure

- `py/macros.py` — MkDocs Macros plugin entry point (`define_env`), deployed as `docs/_utils/py/macros.py` in consumer repos
- `py/pptx_to_reveal.py` — Standalone CLI that converts `.pptx` files to self-contained reveal.js HTML presentations
- `py/requirements.txt` — Python deps for consumer MkDocs projects (`mkdocs`, `mkdocs-material`, `mkdocs-macros-plugin`, `python-pptx`, etc.)
- `js/secrets-client.js` — Browser-side JS client for a local secrets API running at `http://127.0.0.1:9371` (project "coffre-fort")
- `js/mermaid.min.js` — Bundled Mermaid.js library
- `mk/Makefile` — Makefile to be dropped into consumer MkDocs project roots

## Key design points

### macros.py

Registers two MkDocs macros via `define_env(env)`:
- `local_folder(page_url, rootdir)` — renders an HTML `<ul>` of links to files in a folder relative to the current page
- `copy_url_login_mdp(input_category, input_key)` — renders inline links that call `secrets-client.js` functions to copy credentials or open URLs from the local secrets API

### secrets-client.js

Calls `http://127.0.0.1:9371/api/copy` and `/api/get` with query params `category`, `name`, `property`. The server is external (coffre-fort). Functions are called directly from MkDocs page HTML via `onclick` attributes generated by the `copy_url_login_mdp` macro.

### pptx_to_reveal.py

Run as a CLI: `python3 pptx_to_reveal.py [file.pptx ...] [--root <repo-root>]`. Without explicit file arguments it recursively finds all `.pptx` files under `--root`, skipping `.venv`, `site`, and `.git`. Outputs a sibling `.html` file per `.pptx`. Uses `reveal.js` via CDN — no local reveal.js assets needed.

### Makefile (consumer usage)

Expects to be placed at the root of a consumer MkDocs project. Key targets:
```
make activate      # create .venv
make install       # install requirements from docs/_utils/py/requirements.txt
make serve         # mkdocs serve (local dev)
make site-build    # mkdocs build → ./site/
make pptx-convert  # run pptx_to_reveal.py over the whole repo
```
